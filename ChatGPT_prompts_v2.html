Project & Agent Mode Commands

A Practical Cheat-Sheet for Coding, File Management & Automation

Use this guide to operate ChatGPT Projects efficiently ‚Äî including opening files, editing, comparing, running code, debugging, and exporting.

All commands below can be typed directly inside a Project.

Core Commands

Open / View Files

  • "Open core.py." ‚Äî Displays the entire file.
  • "Show me all project files." ‚Äî Lists every file.
  • "Search for the function login_user across all files."

Edit Files

  • "Modify core.py: update the login logic."
  • "Add a new function validate_config() in utils.py."
  • "Fix the bug in line 45 of ui.py."
  • "Refactor database.py to use async."

Code Review / Comparison

  • "Show a diff between old and new core.py."
  • "Explain what changed in ui.py and why."
  • "Compare version1.py and version2.py."

Run Code

  • "Run the project."
  • "Run core.py with sample input."
  • "Call calculate(10, 5) and show output."

Debugging

  • "Run tests and show errors."
  • "Debug the error in data_loader.py."
  • "Trace where this variable becomes None."

File Creation & Management

  • "Create config.json with default settings."
  • "Create README.md describing the project."
  • "Rename old.py to legacy.py."
  • "Delete temp.txt."
  • "Duplicate core.py as core_backup.py."

Import / Export

  • "Export the entire project as a ZIP."
  • "Download core.py."
  • "Upload this file and replace utils.py."

Project Control

  • "Describe the current project structure."
  • "Summarize progress so far and next steps."
  • "Switch to teaching mode: explain changes before saving."
  • "Use fast mode: make changes without long explanations."

What Code Can ChatGPT Run?

ChatGPT Projects execute Python code inside a secure sandbox. Here is a clear overview of what‚Äôs supported, partially supported, or not supported.

‚úîÔ∏è Fully Supported Code

  • Pure Python scripts
  • NumPy, Pandas, Matplotlib
  • SciPy + scikit-learn (common parts)
  • Statistical tests (ANOVA, t-tests, regression, etc.)
  • Data cleaning & transformation
  • File I/O (CSV, JSON, XLSX, images)
  • Pytest test suites
  • Machine learning models (CPU-only)

‚ö†Ô∏è Partially Supported

  • Plotly (static only ‚Äî no interactive UI)
  • Scripts requiring small datasets
  • Long-running tasks (may time out)
  • Code that depends on missing optional packages

‚ùå Not Supported

  • Web servers (Flask, FastAPI, Django)
  • Dashboards (Dash, Streamlit, Gradio)
  • Browser automation (Selenium)
  • Network access / calling external URLs
  • GPU / CUDA code
  • Installing system packages (apt-get)
  • pip install external libraries

üêç Python Version

Projects use a modern Python interpreter (typically 3.9‚Äì3.11). You can verify inside a Project by running:

print(sys.version)

Summary

ChatGPT excels at running analytical Python code, handling files, performing statistics, debugging, and running automated tests. Web servers, dashboards, GPU code, or external connections must run on your local machine ‚Äî but you can still develop and edit them inside a Project.

Syncing Local Code with Uploaded Project Files

ChatGPT Projects keep uploaded files indexed and persistent. If you paste newer code in chat, the model may still rely on older uploaded files unless you synchronize them manually. Use the commands below to ensure your project stays up to date.

‚úîÔ∏è Option 1 ‚Äî Replace Uploaded File with New Code

Paste your updated script and say:

"Replace filename.py with this code:"

This ensures the uploaded version always stays current.

‚úîÔ∏è Option 2 ‚Äî Delete Old Uploaded Files

If you want to work only from chat and avoid file interference:

"Delete file /mnt/data/extendedstat_func_v2.py"

Repeat for any other files you want removed.

‚úîÔ∏è Option 3 ‚Äî Use Uploaded Files as Source of Truth

Keep your authoritative code in the project files and update them incrementally:

"Apply this diff to filename.py"

ChatGPT will update only the changed parts.

‚úîÔ∏è Option 4 ‚Äî Git-Based Workflow

You can ask ChatGPT to maintain Git-style versioning:

"Create a commit message for these changes."

Useful for long-term collaborative projects.

Recommended Workflow

  • Keep master code inside uploaded project files.
  • Always update those files when pasting new code.
  • Avoid mixing outdated uploaded files with new chat code.

Using ChatGPT Memory for Better Coding Sessions

ChatGPT Memory helps the assistant remember your long-term preferences across sessions. While it cannot store code or project files, it can greatly improve how ChatGPT assists you in analysis, code development, and HTML generation.

‚úîÔ∏è Versioned Output Files

Tell ChatGPT to always assume you prefer versioned outputs (e.g. *_v1, *_v2).

"Remember that I always use versioned output files."

This ensures the assistant automatically proposes versioned filenames when generating results.

‚úîÔ∏è Dataset Context

Store the fact that you work with data.

"Remember that I use datasets."

This helps ChatGPT understand your analysis patterns, columns, metrics, and workflows.

‚úîÔ∏è Full Script Generation

Tell ChatGPT to always produce full code scripts rather than partial snippets.

"Remember that I want full code scripts, not partial snippets."

This avoids missing imports, undefined variables, and incomplete refactors.

‚úîÔ∏è High-Quality HTML Formatting

Tell ChatGPT to always maintain clean semantic HTML structure.

"Remember that I want HTML sections well formatted."

Perfect for building cheat sheets, documentation, dashboards, or UI mockups.

‚úîÔ∏è Stored Info

Ask ChatGPT what personal information it currently remembers about you.

"What do you remember about me?"

You can edit or delete stored memory anytime in Settings ‚Üí Personalization ‚Üí Memory.

Recommended Memory Setup

  • Store preferences that affect *how* ChatGPT works, not your code content.
  • Use Memory for workflow and formatting conventions.
  • Do NOT store large code or files in Memory ‚Äî use Project storage.
  • Memory improves consistency across sessions.

Memory Setup Guide

ChatGPT offers two types of memory: Global Memory and Project Context Memory. Understanding their differences helps you create more powerful workflows for coding, documentation, and long-term work inside ChatGPT Projects.

‚úîÔ∏è Global Memory

Controlled by the toggle ‚ÄúReference saved memories‚Äù in Settings.

Use this to store general long-term preferences such as:

  • ‚ÄúRemember that I always use versioned output files.‚Äù
  • ‚ÄúRemember that I want full code scripts, not partial snippets.‚Äù
  • ‚ÄúRemember that I want HTML sections well formatted.‚Äù

These personal preferences apply across all projects and all future conversations.

‚úîÔ∏è Project Context Memory

Automatically enabled and separate from Global Memory.

ChatGPT remembers everything inside each project:

  • Your uploaded files
  • Your Python modules and scripts
  • Your analysis workflow
  • Your error patterns and code structure
  • Your HTML cheat sheet file

‚ùó What Project Context Memory does NOT remember:

  • Your coding style preferences
  • How long or short you want functions to be
  • Your preferred naming conventions
  • Your preferred HTML structure
  • Your preference for full scripts vs partial snippets
  • Any personal preferences about formatting or output

These require Global Memory (Settings ‚Üí Personalization ‚Üí Memory) to persist beyond the current conversation.

Project context is isolated per project ‚Äî your coding project will never interfere with your writing or other domain-specific projects.

‚úîÔ∏è What Memory Cannot Store

For safety, ChatGPT does not store:

  • Your Python source code
  • Your HTML source code
  • Your datasets
  • Your uploaded files

These belong to the Project Storage only. ChatGPT may remember your preferences about how to work with them, but not the files themselves.

‚úîÔ∏è Suggested Memory Commands

These commands help ChatGPT personalize your coding environment:

"Remember that I use datasets."
"Remember that I always want complete code scripts."
"Remember that I prefer versioned output filenames."
"What do you remember about me?"

You can manage or delete saved Memory anytime in Settings ‚Üí Personalization ‚Üí Memory.

Best Practices

  • Use Global Memory for workflow preferences.
  • Rely on Project Context to isolate domains and codebases.
  • Keep your files in the Project folder for persistent access.
  • Use manual upload/replace for files until project editing capabilities improve.
